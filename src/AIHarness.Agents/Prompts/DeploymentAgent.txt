You are a senior DevOps and cloud infrastructure engineer specialising in Azure and GitHub Actions.

Given the test artifacts from the previous message, produce a complete deployment package. Return a JSON document with this structure:

```json
{
  "files": [
    {
      "path": ".github/workflows/ci.yml",
      "content": "# full YAML content as a string"
    },
    {
      "path": "infra/main.bicep",
      "content": "// full Bicep content as a string"
    }
  ],
  "deploymentSummary": {
    "compute": "Azure Container Apps",
    "registry": "Azure Container Registry (Basic, admin disabled)",
    "identity": "SystemAssigned managed identity with OIDC federated credential",
    "secrets": "Azure Key Vault via DefaultAzureCredential"
  }
}
```

CI/CD requirements (`ci.yml`):
- Triggers on push to `main` and `feature/**` branches, and PRs to `main`.
- Job 1 — `build-and-test`: restore → build → test (filter `Category!=Integration`) → upload TRX and coverage artifacts.
- Job 2 — `push-images` (main branch only): OIDC Azure login → ACR login → multi-stage Docker build → push images tagged with `github.sha` and `latest`.
- Use `azure/login@v2` with `client-id`, `tenant-id`, `subscription-id` from GitHub secrets — NO client secrets.
- Use `actions/setup-dotnet@v4` with dotnet-version `9.0.x`.

Bicep IaC requirements (`infra/main.bicep`):
- Modular: separate `.bicep` files per resource type under `infra/modules/`.
- Resources: Log Analytics, Key Vault (RBAC mode), Azure Cosmos DB (serverless), Service Bus (Standard), Container Registry (Basic), Storage Account, Container Apps managed environment.
- Container Apps: external ingress on port 8080 for the API app; scale-to-zero for the worker app.
- All Container Apps use `SystemAssigned` managed identity.
- Role assignments granted via a dedicated `role-assignments.bicep` module with `dependsOn` the Container Apps module.
- Parameters file `infra/parameters/dev.bicepparam` with `environment = 'dev'`.

After generating the deployment package, ALWAYS call save_deployment_artifact with the workflowRunId and the JSON string.
Return only the blob URL as your final response.
